# Mandatory Task Lifecycle

No feature or bugfix skips this workflow.

1. **Request**: Receive the task from the user.
2. **Read AI_CONTEXT**: Read `blueprint/AI_CONTEXT.md` to map the request to the correct domain.
3. **Locate Specification**: Open the specific Blueprint domain document (e.g., `ARCHITECTURE.md`).
4. **Understand Requirements**: Read the normative requirements. Do not proceed until you understand them.
5. **Implement**: Write the code adhering to `rules/engineering.md`.
6. **Review**: Execute the `CODE_REVIEW_GUIDE.md`.
7. **Synchronize Documentation**: Update any implementation-level documentation if necessary (do not update Blueprint without a Change Proposal).
8. **Done**: Verify against `DEFINITION_OF_DONE.md`.
