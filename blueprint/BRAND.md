# Asterim Brand Guidelines

This document outlines the brand identity, positioning, and terminology for Asterim.

## 1. Brand Identity

- **Brand Name**: Asterim
- **Pronunciation**: AS-ter-im
- **Meaning**: Derived from "aster" (star) and "rim" (edge/horizon), representing the frontier of intelligent software engineering.
- **Naming Rationale**: The name evokes a sense of exploration, precision, and the cutting edge of AI, moving away from the more utilitarian "AgentDeck" to something that feels premium, expansive, and visionary.

## 2. Product Positioning

**Official Positioning Statement**:
"Asterim is an AI-native workspace for orchestrating autonomous software engineering agents."

**Core Ideas**:
- **Local-first**: Data and code stay secure on the user's machine.
- **Human-in-the-loop**: Engineers maintain ultimate control and approval over autonomous actions.
- **Multi-agent orchestration**: Coordinating multiple specialized AI agents to solve complex problems.
- **Production-grade software engineering**: Built for real-world, enterprise-level development.
- **Premium developer experience**: Beautiful, responsive, and intuitive interfaces.

## 3. Voice & Tone

- **Authoritative but Accessible**: We speak with confidence about AI and engineering, but avoid overly academic jargon.
- **Empowering**: Our language should make developers feel like they have superpowers.
- **Precise**: We use exact terminology. Agents don't "guess"; they "infer." We don't "run things"; we "orchestrate."
- **Premium**: Communication should feel polished and deliberate, much like the tool itself.

## 4. Product Terminology

| Term | Definition |
|------|------------|
| **Workspace** | The root environment where a project lives. |
| **Agent** | An autonomous or semi-autonomous AI entity capable of executing software engineering tasks. |
| **Orchestration** | The act of coordinating multiple agents and tasks. |
| **Cloud Relay** | The service that connects local Workstations to external or cloud resources. |
| **Workstation** | A local environment where agents operate. |

## 5. Branding Rules

- **Capitalization**: "Asterim" is always capitalized. Never "asterim" unless used in a codebase context (e.g., package names or CLI commands).
- **Logo Usage**: The logo must always have adequate clear space. Do not stretch or distort the logo.
- **No Mixed Branding**: Asterim must never be referred to as "AgentDeck" or "AgentDeck (Asterim)" in user-facing contexts.

## 6. What MUST Never Be Renamed

For backward compatibility and system stability, the following legacy identifiers remain unchanged unless explicitly migrated in a breaking release:
- `.agentdeck` (if used as a user-level configuration directory)
- `AGENTDECK_` prefixed environment variables that are critical for backward compatibility.
- Legacy database migration histories.

## 7. Future Branding Guidelines

As Asterim evolves, any new sub-products or features should follow the same premium, space/exploration-themed naming conventions. Marketing materials should lean into the "frontier of engineering" motif without becoming overly sci-fi.
