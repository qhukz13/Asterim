# Support

Need help with Asterim? We are here to help!

## Getting Help

### 1. GitHub Discussions
For general questions, architecture discussions, feature ideas, and showcase, please use [GitHub Discussions](https://github.com/qhukz13/Asterim/discussions). This is the best place to get help from maintainers and the community.

### 2. GitHub Issues
If you have found a reproducible bug or want to submit a formal feature request, please search our existing [GitHub Issues](https://github.com/qhukz13/Asterim/issues) to see if it has already been reported. If not, feel free to open a new issue using our provided templates.

### 3. Security Vulnerabilities
If you have discovered a security vulnerability, please do **NOT** open a public issue. See our [Security Policy](./SECURITY.md) for instructions on how to report it securely.

## Best Practices for Getting Help

- **Read the Blueprint**: Before asking a design question, please check the `blueprint/` directory. It is our comprehensive source of truth for architecture and product decisions.
- **Provide Context**: When reporting an issue, provide as much context as possible (OS, Node version, logs, and screenshots if applicable).
- **Be Respectful**: Please review and adhere to our [Code of Conduct](./CODE_OF_CONDUCT.md).
